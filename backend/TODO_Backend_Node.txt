================================================================================
  CONCEPTION BASE DE DONNÉES (MongoDB)
================================================================================

------------------------------------------------------------------------
Collection : "batches"
------------------------------------------------------------------------

Schema :
{
  _id          : ObjectId         — identifiant interne MongoDB
  batchId      : string           — UUID v4, retourné immédiatement au client
  status       : enum             — "pending" | "processing" | "completed" | "failed"
  totalDocuments : number         — toujours 1000 dans le sujet
  processedCount : number         — compteur atomique incrémenté par les workers ($inc)
  failedCount  : number           — compteur des échecs définitifs
  userIds      : string[]         — tableau des 1000 userIds reçus
  createdAt    : Date             — géré automatiquement (timestamps: true)
  updatedAt    : Date             — géré automatiquement (timestamps: true)
  completedAt  : Date | null      — rempli quand status passe à "completed"/"failed"
  metadata     : {
    requestIp  : string
    userAgent  : string
  }
}

Index à créer :
  db.batches.createIndex({ batchId: 1 }, { unique: true })
  db.batches.createIndex({ status: 1 })
  db.batches.createIndex({ createdAt: -1 })

------------------------------------------------------------------------
Collection : "documents"
------------------------------------------------------------------------

Schema :
{
  _id              : ObjectId
  documentId       : string         — UUID v4, identifiant public du document
  batchId          : string         — référence vers le batch parent
  userId           : string         — le userId pour lequel le PDF est généré
  status           : enum           — "pending" | "processing" | "completed" | "failed"
  gridfsFileId     : ObjectId|null  — référence vers le fichier dans GridFS
  fileSize         : number|null    — taille en octets du PDF généré
  retryCount       : number         — nombre de tentatives (max 3)
  errorMessage     : string|null    — message d'erreur si échec final
  generationTimeMs : number|null    — durée de génération en ms (observabilité)
  createdAt        : Date
  updatedAt        : Date
}

Index à créer :
  db.documents.createIndex({ documentId: 1 }, { unique: true })
  db.documents.createIndex({ batchId: 1 })
  db.documents.createIndex({ batchId: 1, status: 1 })
  db.documents.createIndex({ userId: 1 })

------------------------------------------------------------------------
GridFS : bucket "pdfs" (collections pdfs.files et pdfs.chunks)
------------------------------------------------------------------------

Chaque fichier stocké avec :
{
  filename      : "${documentId}.pdf"
  contentType   : "application/pdf"
  chunkSizeBytes: 261120              — 255 KB par chunk (défaut GridFS)
  metadata      : {
    documentId  : string
    batchId     : string
    userId      : string
    generatedAt : Date
  }
}

Note : Le bucket s'appelle "pdfs" (pas le bucket "fs" par défaut) pour
       isoler les fichiers PDF des autres éventuels fichiers GridFS.

        const pdfBuffer = await workerPool.runTask({ userId, documentId, batchId })

      ÉTAPE 4 — Uploader vers GridFS en streaming :
        const uploadStream = gridfsStorage.uploadStream(
          `${documentId}.pdf`,
          { documentId, batchId, userId, generatedAt: new Date() }
        )
        Promisifier le stream :
          await new Promise((resolve, reject) => {
            const readable = Readable.from(pdfBuffer)
            readable.pipe(uploadStream)
            uploadStream.on('finish', resolve)
            uploadStream.on('error', reject)
          })

      ÉTAPE 5 — Récupérer l'ID GridFS et la taille :
        const fileId = uploadStream.id
        const fileSize = pdfBuffer.length

      ÉTAPE 6 — Mettre à jour le document en "completed" :
        documentService.updateDocumentStatus(documentId, "completed", {
          gridfsFileId: fileId,
          fileSize,
          generationTimeMs: Date.now() - startTime
        })

      ÉTAPE 7 — Notifier le BatchService :
        batchService.updateBatchProgress(batchId, true)
        logger.info({ documentId, batchId,
                      generationTimeMs: Date.now() - startTime,
                      msg: "PDF généré avec succès" })

      GESTION D'ERREUR (dans le catch) :
        - Logger l'erreur : logger.error({ documentId, batchId, err })
        - Incrémenter retryCount sur le document :
            documentService.updateDocumentStatus(documentId, "failed", {
              errorMessage: err.message,
              retryCount: job.attemptsMade + 1
            })
        - Si job.attemptsMade >= 2 (3ème tentative = finale) :
            Notifier batchService.updateBatchProgress(batchId, false)
        - Relancer l'erreur (throw err) pour que Bull gère le retry

[ ] Implémenter le Graceful Shutdown :
      process.on('SIGTERM', async () => {
        logger.info('SIGTERM reçu, arrêt gracieux en cours...')
        await queueAdapter.close()       — finit les jobs en cours
        await workerPool.destroy()       — attend la fin des threads
        await mongoConnection.disconnect()
        logger.info('Worker arrêté proprement')
        process.exit(0)
      })
